# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="2971a-102">Derleyici Uyarısı (Düzey 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="2971a-102">Compiler Warning (level 3) CS0414</span></span>

 <span data-ttu-id="2971a-104">Bu uyarı, derleyici bir değişkeni hiçbir zaman başvurulan doğrulayabilirsiniz çeşitli senaryolarda oluşabilir:</span><span class="sxs-lookup"><span data-stu-id="2971a-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>

-   <span data-ttu-id="2971a-112">Özel veya iç alan birden fazla konumda sabit değerler atanır ancak aksi erişilebilir.</span><span class="sxs-lookup"><span data-stu-id="2971a-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="2971a-113">Alan gerekmiyorsa kaldırmadan göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="2971a-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="2971a-114">Aksi takdirde, uygun şekilde kullanın.</span><span class="sxs-lookup"><span data-stu-id="2971a-114">Otherwise, use it in some appropriate way.</span></span>

 <span data-ttu-id="2971a-116">Aşağıdaki örnek CS0414 oluşturulacak yollarından biri gösterilmektedir:</span><span class="sxs-lookup"><span data-stu-id="2971a-116">The following sample shows one way in which CS0414 will be generated:</span></span>

{  
   private int i = 1;  // CS0414
