# <a name="compiler-warning-level-1-cs4014"></a><span data-ttu-id="1daa8-102">Derleyici Uyarısı (düzey 1) CS4014</span><span class="sxs-lookup"><span data-stu-id="1daa8-102">Compiler Warning (level 1) CS4014</span></span>

<span data-ttu-id="1daa8-103">Bu çağrı beklenmediği için, geçerli yöntemin yürütülmesi çağrı tamamlanmadan önce devam ediyor.</span><span class="sxs-lookup"><span data-stu-id="1daa8-103">Because this call is not awaited, execution of the current method continues before the call is completed.</span></span> <span data-ttu-id="1daa8-104">'Bekleme' işlecinin arama sonucuna başvurmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="1daa8-104">Consider applying the 'await' operator to the result of the call.</span></span>  
  
<span data-ttu-id="1daa8-105">Geçerli yöntem, bir veya bir <xref:System.Threading.Tasks.Task> <xref:System.Threading.Tasks.Task%601> döndüren ve [bekleyen](../operators/await.md) işleci sonuca uygulamayan bir async yöntemi çağırır.</span><span class="sxs-lookup"><span data-stu-id="1daa8-105">The current method calls an async method that returns a <xref:System.Threading.Tasks.Task> or a <xref:System.Threading.Tasks.Task%601> and doesn’t apply the [await](../operators/await.md) operator to the result.</span></span> <span data-ttu-id="1daa8-106">Async yöntemine çağrı asynchronous bir görev başlatır.</span><span class="sxs-lookup"><span data-stu-id="1daa8-106">The call to the async method starts an asynchronous task.</span></span> <span data-ttu-id="1daa8-107">Ancak, operatör `await` uygulanmadığından, program görevin tamamlanmasını beklemeden devam eder.</span><span class="sxs-lookup"><span data-stu-id="1daa8-107">However, because no `await` operator is applied, the program continues without waiting for the task to complete.</span></span> <span data-ttu-id="1daa8-108">Çoğu durumda, bu davranış beklediğiniz gibi değildir.</span><span class="sxs-lookup"><span data-stu-id="1daa8-108">In most cases, that behavior isn't what you expect.</span></span> <span data-ttu-id="1daa8-109">Genellikle arama yönteminin diğer yönleri aramanın sonuçlarına bağlıdır veya en az olarak, çağrıyı içeren yöntemden dönmeden önce çağrı yönteminin tamamlanması beklenir.</span><span class="sxs-lookup"><span data-stu-id="1daa8-109">Usually other aspects of the calling method depend on the results of the call or, minimally, the called method is expected to complete before you return from the method that contains the call.</span></span>  
  
<span data-ttu-id="1daa8-110">Aynı derecede önemli bir konu, adverilen async yönteminde yükseltilen özel durumlara ne olduğudur.</span><span class="sxs-lookup"><span data-stu-id="1daa8-110">An equally important issue is what happens to exceptions that are raised in the called async method.</span></span> <span data-ttu-id="1daa8-111">Döndürülen görevde bir <xref:System.Threading.Tasks.Task> döndüren veya <xref:System.Threading.Tasks.Task%601> depolanan bir yöntemde yükseltilen bir özel durum.</span><span class="sxs-lookup"><span data-stu-id="1daa8-111">An exception that's raised in a method that returns a <xref:System.Threading.Tasks.Task> or  <xref:System.Threading.Tasks.Task%601> is stored in the returned task.</span></span> <span data-ttu-id="1daa8-112">Görevi beklemezseniz veya özel durumları açıkça denetlerseniz, özel durum kaybolur.</span><span class="sxs-lookup"><span data-stu-id="1daa8-112">If you don't await the task or explicitly check for exceptions, the exception is lost.</span></span> <span data-ttu-id="1daa8-113">Görevi bekliyorsanız, özel durumu yeniden atılır.</span><span class="sxs-lookup"><span data-stu-id="1daa8-113">If you await the task, its exception is rethrown.</span></span>  

<span data-ttu-id="1daa8-114">En iyi uygulama olarak, her zaman arama beklemelisiniz.</span><span class="sxs-lookup"><span data-stu-id="1daa8-114">As a best practice, you should always await the call.</span></span>

<span data-ttu-id="1daa8-115">Uyarıyı yalnızca eşzamanlı aramanın tamamlanmasını beklemek istemediğiniz ve çağrılan yöntemin herhangi bir özel durum doğurmayacaklarından eminseniz uyarıyı bastırmayı düşünmelisiniz.</span><span class="sxs-lookup"><span data-stu-id="1daa8-115">You should consider suppressing the warning only if you're sure that you don't want to wait for the asynchronous call to complete and that the called method won't raise any exceptions.</span></span> <span data-ttu-id="1daa8-116">Bu durumda, çağrının görev sonucunu bir değişkene atayarak uyarıyı bastırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="1daa8-116">In that case, you can suppress the warning by assigning the task result of the call to a variable.</span></span>  

<span data-ttu-id="1daa8-117">Aşağıdaki örnek, uyarının nasıl neden olacağını, nasıl bastırılabildiğini ve aramayı nasıl bekleyeceğini gösterir.</span><span class="sxs-lookup"><span data-stu-id="1daa8-117">The following example shows how to cause the warning, how to suppress it, and how to await the call.</span></span>  

```csharp
async Task CallingMethodAsync()  
{  
    resultsTextBox.Text += "\r\n  Entering calling method.";  
    // Variable delay is used to slow down the called method so that you can  
    // distinguish between awaiting and not awaiting in the program's output.  
    // You can adjust the value to produce the output that this topic shows  
    // after the code.  
    var delay = 5000;  
  
    // Call #1.  
    // Call an async method. Because you don't await it, its completion
    // isn't coordinated with the current method, CallingMethodAsync.  
    // The following line causes warning CS4014.  
    CalledMethodAsync(delay);  
  
    // Call #2.  
    // To suppress the warning without awaiting, you can assign the
    // returned task to a variable. The assignment doesn't change how  
    // the program runs. However, recommended practice is always to  
    // await a call to an async method.  
  
    // Replace Call #1 with the following line.  
    //Task delayTask = CalledMethodAsync(delay);  
  
    // Call #3  
    // To contrast with an awaited call, replace the unawaited call
    // (Call #1 or Call #2) with the following awaited call. Best
    // practice is to await the call.  
  
    //await CalledMethodAsync(delay);  
  
    // If the call to CalledMethodAsync isn't awaited, CallingMethodAsync  
    // continues to run and, in this example, finishes its work and returns  
    // to its caller.  
    resultsTextBox.Text += "\r\n  Returning from calling method.";  
}  
  
async Task CalledMethodAsync(int howLong)  
{  
    resultsTextBox.Text +=
        "\r\n    Entering called method, starting and awaiting Task.Delay.";  
  
    // Slow the process down a little so that you can distinguish between  
    // awaiting and not awaiting in the program's output. Adjust the value  
    // for howLong if necessary.  
    await Task.Delay(howLong);  
    resultsTextBox.Text +=
        "\r\n    Task.Delay is finished--returning from called method.";  
}  
```

<span data-ttu-id="1daa8-118">Örnekte, #1 arama veya #2 Arama'yı seçerseniz, beklenen`CalledMethodAsync`beklemesiz async yöntemi`CallingMethodAsync`( ) hem arayanın ( ) hem de arayanın arayan ( )`startButton_Click`tamamlandıktan sonra tamamlanır.</span><span class="sxs-lookup"><span data-stu-id="1daa8-118">In the example, if you choose Call #1 or Call #2, the unawaited async method (`CalledMethodAsync`) finishes after both its caller (`CallingMethodAsync`) and the caller's caller (`startButton_Click`) are complete.</span></span> <span data-ttu-id="1daa8-119">Aşağıdaki çıktıdaki son satır, çağrılan yöntem bittiğinde size gösterir.</span><span class="sxs-lookup"><span data-stu-id="1daa8-119">The last line in the following output shows you when the called method finishes.</span></span> <span data-ttu-id="1daa8-120">Tam örnekte arama `CallingMethodAsync` yapan olay işleyicisine giriş ve çıkış çıktı çıktısında işaretlenir.</span><span class="sxs-lookup"><span data-stu-id="1daa8-120">Entry to and exit from the event handler that calls `CallingMethodAsync` in the full example are marked in the output.</span></span>
