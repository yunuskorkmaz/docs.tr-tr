## <a name="example"></a><span data-ttu-id="b6ec0-108">Örnek</span><span class="sxs-lookup"><span data-stu-id="b6ec0-108">Example</span></span>  
 <span data-ttu-id="b6ec0-109">Aşağıdaki örnek CS0269 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="b6ec0-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="b6ec0-110">Örnek</span><span class="sxs-lookup"><span data-stu-id="b6ec0-110">Example</span></span>  
 <span data-ttu-id="b6ec0-111">Bu durum, bir değişkenin başlatılması bir try bloğunda gerçekleşirse, derleyicinin doğrulayamaması durumunda da meydana gelir:</span><span class="sxs-lookup"><span data-stu-id="b6ec0-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
