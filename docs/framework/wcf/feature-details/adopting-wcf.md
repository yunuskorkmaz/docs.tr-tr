# <a name="adopt-windows-communication-foundation"></a><span data-ttu-id="594d8-102">Windows Communication Foundation benimseyin</span><span class="sxs-lookup"><span data-stu-id="594d8-102">Adopt Windows Communication Foundation</span></span>

<span data-ttu-id="594d8-103">Yeni geliştirme için Windows Communication Foundation (WCF) kullanmayı seçebilirsiniz ve ASP.NET kullanarak geliştirilen mevcut uygulamaları sürdürmeye devam edebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="594d8-103">You can choose to use Windows Communication Foundation (WCF) for new development while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="594d8-104">WCF 'nin herhangi bir senaryoda .NET Framework ile oluşturulmuş uygulamalarla iletişimi kolaylaştırmak için en uygun seçim olması amaçlandığından, çok çeşitli yazılım iletişimleri sorunlarını ASP.NET bir şekilde çözmek için standart bir araç işlevi görebilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="594d8-105">Yeni WCF uygulamaları, var olan ASP.NET Web hizmetleriyle aynı makinelere dağıtılabilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="594d8-106">Mevcut ASP.NET Web Hizmetleri sürüm 2,0 ' den önceki bir .NET Framework sürümünü kullanıyorsa, yeni WCF uygulamalarının barındırıldığı IIS uygulamalarına .NET Framework 2,0 ' i seçmeli olarak dağıtmak için ASP.NET IIS kayıt aracı 'nı kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="594d8-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="594d8-107">Bu araç [ASP.NET IIS Kayıt Aracı (Aspnet_regiis.exe)](/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90))adresinde BELGELENMIŞTIR ve IIS 6,0 yönetim konsoluna yerleşik bir kullanıcı arabirimine sahiptir.</span><span class="sxs-lookup"><span data-stu-id="594d8-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90)), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="594d8-108">WCF, ASP.NET uyumluluk modunda çalışmak üzere yapılandırılmış WCF hizmetlerini IIS 'de var olan ASP.NET Web hizmeti uygulamalarına ekleyerek mevcut ASP.NET Web hizmetlerine yeni özellikler eklemek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="594d8-109">ASP.NET uyumluluk modu nedeniyle, yeni WCF hizmetlerinin kodu, sınıfı kullanılarak önceden var olan ASP.NET kodu ile aynı uygulama durumu bilgilerine erişebilir ve güncelleştirebilir <xref:System.Web.HttpContext> .</span><span class="sxs-lookup"><span data-stu-id="594d8-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="594d8-110">Uygulamalar aynı sınıf kitaplıklarını da paylaşabilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="594d8-111">WCF istemcileri, ASP.NET Web Hizmetleri 'ni kullanabilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="594d8-112">İle yapılandırılan WCF Hizmetleri, <xref:System.ServiceModel.BasicHttpBinding> ASP.NET Web hizmeti istemcileri tarafından kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="594d8-113">ASP.NET Web Hizmetleri WCF uygulamalarıyla birlikte çalışabilir ve WCF, var olan ASP.NET Web hizmetlerine özellikler eklemek için bile kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-113">ASP.NET Web services can coexist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="594d8-114">WCF ve ASP.NET Web hizmetlerinin birlikte kullanılabileceği tüm bu yollar verildiğinde, ASP.NET Web Hizmetleri 'ni yalnızca WCF tarafından sağlanan ve ASP.NET Web Hizmetleri olmayan özelliklere ihtiyacınız varsa WCF 'ye geçirmek isteyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="594d8-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="594d8-115">Gerekli olduğu durumlarda bile, kodun bir teknolojiden diğerine geçirilmesi nadiren doğru yaklaşımda olur.</span><span class="sxs-lookup"><span data-stu-id="594d8-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="594d8-116">Yeni teknolojinin benimseme nedeni, önceki teknolojiyle karşılanmayacak yeni gereksinimleri karşılamadır ve bu durumda, doğru olan şey yeni bir çözümü yeni bir çözüm tasarlayarak yeni bir çözüm tasarlamadır.</span><span class="sxs-lookup"><span data-stu-id="594d8-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly expanded set of requirements.</span></span> <span data-ttu-id="594d8-117">Mevcut sistemle ve bu sistem tasarlanmasından bu yana elde edilen wısystem ile deneyiminizden yeni tasarım avantajları.</span><span class="sxs-lookup"><span data-stu-id="594d8-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="594d8-118">Yeni tasarım, yeni teknolojiden eski tasarımın yerine yeni teknolojilerin tam yeteneklerini de kullanabilir.</span><span class="sxs-lookup"><span data-stu-id="594d8-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="594d8-119">Yeni tasarımın anahtar öğelerini prototipten sonra, yeni bir sistemdeki kodu yeniden kullanmak daha kolay hale gelir.</span><span class="sxs-lookup"><span data-stu-id="594d8-119">After prototyping key elements of the new design, it becomes easier to reuse code from the existing system within the new one.</span></span>
