-   <span data-ttu-id="62f51-107">İzleme çıkışı için bir hedef belirtin.</span><span class="sxs-lookup"><span data-stu-id="62f51-107">Specify a destination for tracing output.</span></span>  
  
-   <span data-ttu-id="62f51-108">Ağ izleme davranışını yapılandırın.</span><span class="sxs-lookup"><span data-stu-id="62f51-108">Configure the behavior of network tracing.</span></span> <span data-ttu-id="62f51-109">Bkz: [nasıl yapılır: ağ izlemeyi yapılandırma](../../../docs/framework/network-programming/how-to-configure-network-tracing.md) ayrıntılı bilgi için.</span><span class="sxs-lookup"><span data-stu-id="62f51-109">See [How to: Configure Network Tracing](../../../docs/framework/network-programming/how-to-configure-network-tracing.md) for detailed information.</span></span>  
  
 <span data-ttu-id="62f51-110">İzleme dinleyicileri da adlandırılan en yaygın izleme hedefleri şunlardır: varsayılan dinleyici ve günlük dosyası.</span><span class="sxs-lookup"><span data-stu-id="62f51-110">The most common trace destinations, also referred to as trace listeners, are the default listener and the log file.</span></span>  
  
 <span data-ttu-id="62f51-111">İzleme, İzleme dinleyicisi belirtmezseniz varsayılan dinleyicisi kullanır.</span><span class="sxs-lookup"><span data-stu-id="62f51-111">Tracing uses the default listener if you do not specify a trace listener.</span></span> <span data-ttu-id="62f51-112">CLR hata ayıklayıcısı .NET Framework SDK ile birlikte gelen veya DBwin32.exe Windows SDK'sı ile birlikte gelen gibi bir yönetilen kod etkin hata ayıklayıcıda kodunuzu yürüterek varsayılan dinleyiciye gönderilen iletileri görüntüleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="62f51-112">You can view messages sent to the default listener by executing your code in a managed code-enabled debugger such as the CLR Debugger shipped with the .NET Framework SDK, or DBwin32.exe shipped with the Windows SDK.</span></span> <span data-ttu-id="62f51-113">CLR hata ayıklayıcıyı kullanarak izleme iletilerini görünür **çıkış** penceresi.</span><span class="sxs-lookup"><span data-stu-id="62f51-113">Using the CLR Debugger, the trace messages appear in the **Output** window.</span></span>  
  
 <span data-ttu-id="62f51-114">İzlemeleri almak için bir dosya kullanmayı tercih ederseniz, günlük dosyası, yapılandırma ayarlarını kullanarak aşağıdaki örnekte gösterildiği gibi belirtebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="62f51-114">If you prefer to use a file to receive traces, you can specify a log file by using configuration settings, as shown in the following example.</span></span> <span data-ttu-id="62f51-115">(Yapılandırma dosyaları hakkında genel bilgiler için bkz: [yapılandırma dosyalarını](../../../docs/framework/configure-apps/index.md).)</span><span class="sxs-lookup"><span data-stu-id="62f51-115">(For a general discussion of configuration files, see [Configuration Files](../../../docs/framework/configure-apps/index.md).)</span></span>  
  
 <span data-ttu-id="62f51-116">Bir günlük dosyasına izlemeleri göndermek için aşağıdaki düğüm için ekleme `<system.diagnostics>` düğümü uygun yapılandırma dosyasının (uygulama veya makine).</span><span class="sxs-lookup"><span data-stu-id="62f51-116">To send traces to a log file, add the following node to the `<system.diagnostics>` node of the appropriate configuration file (application or machine).</span></span> <span data-ttu-id="62f51-117">İhtiyaçlarınıza uygun şekilde (trace.log) dosyasının adını değiştirebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="62f51-117">You can change the name of the file (trace.log) to suit your needs.</span></span>  
  
```xml

  <trace autoflush="true" indentsize="4">  
    <listeners>  
      <add name="file" type="System.Diagnostics.TextWriterTraceListener" initializeData="trace.log"/>  
    </listeners>   
  </trace>  
</system.diagnostics>  
```
