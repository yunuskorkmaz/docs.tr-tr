 <span data-ttu-id="edfa7-119">Aslında, Kullanıcı ölçek faktörünü 120 dpi olarak ayarladığında, ekrandaki dikey veya yatay inç yüzde 25 oranında daha büyük olur.</span><span class="sxs-lookup"><span data-stu-id="edfa7-119">In effect, when the user sets the scale factor to 120 dpi, a vertical or horizontal inch on the screen becomes bigger by 25 percent.</span></span> <span data-ttu-id="edfa7-120">Tüm boyutlar uygun şekilde ölçeklendirilir.</span><span class="sxs-lookup"><span data-stu-id="edfa7-120">All dimensions are scaled accordingly.</span></span> <span data-ttu-id="edfa7-121">Ekranın üst ve sol kenarlarından bir uygulama penceresinin kayması yüzde 25 oranında artar.</span><span class="sxs-lookup"><span data-stu-id="edfa7-121">The offset of an application window from the top and left edges of the screen increases by 25 percent.</span></span> <span data-ttu-id="edfa7-122">Uygulama ölçekleme etkinse ve uygulama DPI duyarlı değilse, pencerenin boyutu, içerdiği tüm öğelerin uzaklıkları ve boyutları ile aynı oranda artar [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] .</span><span class="sxs-lookup"><span data-stu-id="edfa7-122">If application scaling is enabled and the application is not dpi-aware, the size of the window increases in the same proportion, along with the offsets and sizes of all [!INCLUDE[TLA2#tla_ui](../../../includes/tla2sharptla-ui-md.md)] elements it contains.</span></span>

 <span data-ttu-id="edfa7-125">Ekran ölçeklendirme, ekran koordinatları ile herhangi bir şekilde ilgilenen uygulamalar için yeni zorluk oluşturur.</span><span class="sxs-lookup"><span data-stu-id="edfa7-125">Screen scaling creates new challenges for applications that are concerned in any way with screen coordinates.</span></span> <span data-ttu-id="edfa7-126">Ekranda şu anda iki koordinat sistemi var: fiziksel ve mantıksal.</span><span class="sxs-lookup"><span data-stu-id="edfa7-126">The screen now contains two coordinate systems: physical and logical.</span></span> <span data-ttu-id="edfa7-127">Bir noktanın fiziksel koordinatları, kaynağın sol üst kısmından itibaren gerçek uzaklığa göre piksel cinsinden fark edilir.</span><span class="sxs-lookup"><span data-stu-id="edfa7-127">The physical coordinates of a point are the actual offset in pixels from the top left of the origin.</span></span> <span data-ttu-id="edfa7-128">Mantıksal Koordinatlar, piksellerin kendisi ölçeklendiği gibi uzaklıklardır.</span><span class="sxs-lookup"><span data-stu-id="edfa7-128">The logical coordinates are the offsets as they would be if the pixels themselves were scaled.</span></span>

 <span data-ttu-id="edfa7-133">Mantıksal Koordinatlar, DPI ayarından bağımsız olarak işletim sisteminin ve uygulamaların davranışını tutarlı hale yaptığından önemlidir.</span><span class="sxs-lookup"><span data-stu-id="edfa7-133">Logical coordinates are important, because they make the behavior of the operating system and applications consistent regardless of the dpi setting.</span></span> <span data-ttu-id="edfa7-134">Örneğin, <xref:System.Windows.Forms.Cursor.Position%2A?displayProperty=nameWithType> normalde mantıksal koordinatları döndürür.</span><span class="sxs-lookup"><span data-stu-id="edfa7-134">For example, <xref:System.Windows.Forms.Cursor.Position%2A?displayProperty=nameWithType> normally returns the logical coordinates.</span></span> <span data-ttu-id="edfa7-135">İmleci bir iletişim kutusunda bir öğenin üzerine taşırsanız, DPI ayarından bağımsız olarak aynı koordinatlar döndürülür.</span><span class="sxs-lookup"><span data-stu-id="edfa7-135">If you move the cursor over an element in a dialog box, the same coordinates are returned regardless of the dpi setting.</span></span> <span data-ttu-id="edfa7-136">(100, 100) adresinde bir Denetim çiziyorsanız, bu mantıksal koordinatlara çizilir ve herhangi bir DPI ayarında aynı göreli konumu kaplayacaktır.</span><span class="sxs-lookup"><span data-stu-id="edfa7-136">If you draw a control at (100, 100), it is drawn to those logical coordinates, and will occupy the same relative position at any dpi setting.</span></span>
