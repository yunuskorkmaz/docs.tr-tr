## <a name="implementation-guidelines-and-conventions"></a><span data-ttu-id="aebcf-109">Uygulama yönergeleri ve kuralları</span><span class="sxs-lookup"><span data-stu-id="aebcf-109">Implementation Guidelines and Conventions</span></span>  
 <span data-ttu-id="aebcf-110">Dönüşüm denetim düzeni uygularken aşağıdaki yönergeler ve kuralları dikkat edin:</span><span class="sxs-lookup"><span data-stu-id="aebcf-110">When implementing the Transform control pattern, note the following guidelines and conventions:</span></span>  
  
-   <span data-ttu-id="aebcf-111">Bu denetim düzeni için destek masaüstünde nesnelere sınırlı değildir.</span><span class="sxs-lookup"><span data-stu-id="aebcf-111">Support for this control pattern is not limited to objects on the desktop.</span></span> <span data-ttu-id="aebcf-112">Alt taşınmış, yeniden boyutlandırılabilir veya serbestçe kapsayıcının sınırları içinde döndürülen, bu denetim düzeni de kapsayıcı nesnesinin alt tarafından desteklenmesi gerekir.</span><span class="sxs-lookup"><span data-stu-id="aebcf-112">This control pattern must also be supported by the children of a container object if the children can be moved, resized, or rotated freely within the boundaries of the container.</span></span>  
  
-   <span data-ttu-id="aebcf-113">Bir nesne taşınamaz, yeniden boyutlandırılmış veya elde edilen ekran konumunu veya tamamen kapsayıcısının ve bu nedenle erişilemez klavye koordinatları dışında olması (örneğin, bir üst düzey pencere off-screen taşındığında veya bir fare Döndürülmüş alt nesne kapsayıcının Görünüm penceresi sınırları dışında taşınır).</span><span class="sxs-lookup"><span data-stu-id="aebcf-113">An object cannot be moved, resized, or rotated such that its resulting screen location would be completely outside the coordinates of its container and therefore inaccessible to the keyboard or mouse (for example, when a top-level window is moved off-screen or a child object is moved outside the boundaries of the container's viewport).</span></span> <span data-ttu-id="aebcf-114">Bu gibi durumlarda, istenen ekran koordinatları olarak mümkün olduğunca kapsayıcı sınırlar içinde geçersiz üst veya sol koordinatları ile yakın nesne yerleştirilir.</span><span class="sxs-lookup"><span data-stu-id="aebcf-114">In these cases, the object is placed as close to the requested screen coordinates as possible with the top or left coordinates overridden to be within the container boundaries.</span></span>  
  
-   <span data-ttu-id="aebcf-115">Bir nesne taşınırsa, çoklu monitör sistemler için yeniden boyutlandırılıyor veya tamamen birleşik masaüstü ekran koordinatları dışında döndürülen nesne birincil monitörde istenen koordinatları olarak mümkün olduğunca yakın yerleştirilir.</span><span class="sxs-lookup"><span data-stu-id="aebcf-115">For multi-monitor systems, if an object is moved, resized, or rotated completely outside the combined desktop screen coordinates, the object is placed on the primary monitor as close to the requested coordinates as possible.</span></span>  
  
-   <span data-ttu-id="aebcf-116">Tüm parametrelerin ve özellik değerlerini mutlak ve yerel ayar bağımsızdır.</span><span class="sxs-lookup"><span data-stu-id="aebcf-116">All parameters and property values are absolute and independent of locale.</span></span>
