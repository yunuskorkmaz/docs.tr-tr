<span data-ttu-id="7ab89-107">Daha önce bu e-kitap, sunduk *kümeleri* ve *zamanlayıcılar* yazılım mimarisi ve geliştirme tartışmaya bir parçası olarak.</span><span class="sxs-lookup"><span data-stu-id="7ab89-107">Earlier in this e-book, we introduced *clusters* and *schedulers* as part of the discussion on software architecture and development.</span></span> <span data-ttu-id="7ab89-108">Docker Swarm ve Mesosphere Datacenter işletim sistemini (DC/OS) Docker kümeleri örnekleridir.</span><span class="sxs-lookup"><span data-stu-id="7ab89-108">Examples of Docker clusters are Docker Swarm and Mesosphere Datacenter Operating System (DC/OS).</span></span> <span data-ttu-id="7ab89-109">Bunların her ikisi de, Microsoft Azure Container Service tarafından sunulan altyapıyı bir parçası olarak çalıştırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="7ab89-109">Both of these can run as a part of the infrastructure provided by Microsoft Azure Container Service.</span></span>

<span data-ttu-id="7ab89-110">Uygulamalar arasında birden fazla konak sistemi genişletilmiş olduğunda, her bir konak sistemi yönetmek ve temel alınan platformu karmaşıklığını hemen soyut olanağı cazip hale gelir.</span><span class="sxs-lookup"><span data-stu-id="7ab89-110">When applications are scaled-out across multiple host systems, the ability to manage each host system and abstract away the complexity of the underlying platform becomes attractive.</span></span> <span data-ttu-id="7ab89-111">Tam olarak neler düzenleyiciler ve zamanlayıcılar sağlar olmasıdır.</span><span class="sxs-lookup"><span data-stu-id="7ab89-111">That is precisely what orchestrators and schedulers provide.</span></span> <span data-ttu-id="7ab89-112">Burada bunları kısa bir göz atalım:</span><span class="sxs-lookup"><span data-stu-id="7ab89-112">Let's take a brief look at them here:</span></span>

- <span data-ttu-id="7ab89-113">**Zamanlayıcılar**.</span><span class="sxs-lookup"><span data-stu-id="7ab89-113">**Schedulers**.</span></span><span data-ttu-id="7ab89-114"> "Zamanlama" özelliği bir yöneticinin hizmet dosya belirli bir kapsayıcı çalıştırma kurar ve bir konak sistemi yüklemesini ifade eder.</span><span class="sxs-lookup"><span data-stu-id="7ab89-114"> "Scheduling" refers to the ability for an administrator to load a service file onto a host system that establishes how to run a specific container.</span></span> <span data-ttu-id="7ab89-115">Kapsayıcıları bir Docker kümesi başlatma zamanlama olarak bilinen eğilimindedir.</span><span class="sxs-lookup"><span data-stu-id="7ab89-115">Launching containers in a Docker cluster tends to be known as scheduling.</span></span> <span data-ttu-id="7ab89-116">Hizmet tanımında daha genel bir fikir yükleme belirli Yasası zamanlama başvuruyor ancak zamanlayıcılar gereken herhangi bir kapasite hizmetleri yönetmek için bir ana bilgisayarın init sisteme takma için sorumlu olursunuz.</span><span class="sxs-lookup"><span data-stu-id="7ab89-116">Although scheduling refers to the specific act of loading the service definition, in a more general sense, schedulers are responsible for hooking into a host's init system to manage services in whatever capacity needed.</span></span>

   <span data-ttu-id="7ab89-117">Bir küme Zamanlayıcı birden çok hedefi vardır: küme kaynaklarını verimli bir şekilde kullanarak, kullanıcı tarafından sağlanan yerleştirme kısıtlamaları, "eşitliği, hataları güçlü olan" derecesini sahip hızlı bir şekilde onlara bir bekleyen durumda bırakmamaya uygulamalar planlama ile çalışma ve her zaman kullanılabilir olacak.</span><span class="sxs-lookup"><span data-stu-id="7ab89-117">A cluster scheduler has multiple goals: using the cluster's resources efficiently, working with user-supplied placement constraints, scheduling applications rapidly to not leave them in a pending state, having a degree of "fairness," being robust to errors, and always be available.</span></span>

- <span data-ttu-id="7ab89-118">**Orchestration**.</span><span class="sxs-lookup"><span data-stu-id="7ab89-118">**Orchestration**.</span></span><span data-ttu-id="7ab89-119"> Platformları, bir konak kümesi üzerinde dağıtılmış karmaşık, çok kapsayıcılı iş yüklerini yaşam döngüsü yönetim özelliklerini genişletin.</span><span class="sxs-lookup"><span data-stu-id="7ab89-119"> Platforms extend life-cycle management capabilities to complex, multicontainer workloads deployed on a cluster of hosts.</span></span> <span data-ttu-id="7ab89-120">Aldığı konak altyapısının özetleyen tarafından düzenleme araçları kullanıcıların tüm küme tek dağıtım hedefi olarak değerlendirilecek bir yol sağlar.</span><span class="sxs-lookup"><span data-stu-id="7ab89-120">By abstracting the host infrastructure, orchestration tools give users a way to treat the entire cluster as a single deployment target.</span></span>

   <span data-ttu-id="7ab89-122">Orchestration kapsayıcı zamanlama, küme yönetimi ve büyük olasılıkla ek ana sağlama için başvuruda bulunan bir geniş bir terimdir.</span><span class="sxs-lookup"><span data-stu-id="7ab89-122">Orchestration is a broad term that refers to container scheduling, cluster management, and possibly the provisioning of additional hosts.</span></span>
