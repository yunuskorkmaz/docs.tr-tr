# <a name="choosing-between-class-and-struct"></a><span data-ttu-id="8df5a-102">Sınıf ile yapı arasında seçim yapma</span><span class="sxs-lookup"><span data-stu-id="8df5a-102">Choosing Between Class and Struct</span></span>
<span data-ttu-id="8df5a-103">Her framework Tasarımcısı yüz temel tasarım kararlarından biri mi (bir başvuru türü) bir sınıf veya yapı (bir değer türü) olarak bir tür tasarlamak biridir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-103">One of the basic design decisions every framework designer faces is whether to design a type as a class (a reference type) or as a struct (a value type).</span></span> <span data-ttu-id="8df5a-104">Başvuru türleri ve değer türlerinin davranış farklılıkları iyi anlaşılmasını, bu seçim yaparken önemlidir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-104">Good understanding of the differences in the behavior of reference types and value types is crucial in making this choice.</span></span>  
  
 <span data-ttu-id="8df5a-105">İlk yığın başvuru türleri ve değer türleri biz başvuru türleri ise, değer türleri yığın üzerinde ayrılan veya içeren satır içi türleri ve ne zaman serbest ve atık toplanan yığında ayrılmış olmasıdır dikkate alacaktır fark geriye doğru alır veya ne zaman, içeren türlerine serbest bırakıldı.</span><span class="sxs-lookup"><span data-stu-id="8df5a-105">The first difference between reference types and value types we will consider is that reference types are allocated on the heap and garbage-collected, whereas value types are allocated either on the stack or inline in containing types and deallocated when the stack unwinds or when their containing type gets deallocated.</span></span> <span data-ttu-id="8df5a-106">Bu nedenle, genel ayırma ve başvuru türleri deallocations ucuz ayırmaları ve değer türlerinin deallocations altındadır.</span><span class="sxs-lookup"><span data-stu-id="8df5a-106">Therefore, allocations and deallocations of value types are in general cheaper than allocations and deallocations of reference types.</span></span>  
  
 <span data-ttu-id="8df5a-107">Ardından, diziler başvuru türleridir çıkış dizi anlamı satır dışı, yalnızca başvuru yığında bulunan başvuru türünün örneğini tahsis edilir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-107">Next, arrays of reference types are allocated out-of-line, meaning the array elements are just references to instances of the reference type residing on the heap.</span></span> <span data-ttu-id="8df5a-108">Değer türü diziler, dizi öğelerinin değer türü gerçek örneklerini olduğu anlamına satır içi, ayrılır.</span><span class="sxs-lookup"><span data-stu-id="8df5a-108">Value type arrays are allocated inline, meaning that the array elements are the actual instances of the value type.</span></span> <span data-ttu-id="8df5a-109">Bu nedenle, ayırma ve değer türü diziler deallocations ayırmaları ve başvuru türü diziler deallocations ucuz.</span><span class="sxs-lookup"><span data-stu-id="8df5a-109">Therefore, allocations and deallocations of value type arrays are much cheaper than allocations and deallocations of reference type arrays.</span></span> <span data-ttu-id="8df5a-110">Ayrıca, çoğu durumda, değer türü diziler çok daha iyi bir yerleşim yeri başvuru sergiler.</span><span class="sxs-lookup"><span data-stu-id="8df5a-110">In addition, in a majority of cases value type arrays exhibit much better locality of reference.</span></span>  
  
 <span data-ttu-id="8df5a-111">Sonraki farka bellek kullanımı ile ilgilidir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-111">The next difference is related to memory usage.</span></span> <span data-ttu-id="8df5a-112">Değer türleri, bir başvuru türü veya uyguladıkları arabirimlerinden birini olduğunda cast Kutulu.</span><span class="sxs-lookup"><span data-stu-id="8df5a-112">Value types get boxed when cast to a reference type or one of the interfaces they implement.</span></span> <span data-ttu-id="8df5a-113">Kutulanmamış aldıkları ne zaman değer türüne dönüştürme.</span><span class="sxs-lookup"><span data-stu-id="8df5a-113">They get unboxed when cast back to the value type.</span></span> <span data-ttu-id="8df5a-114">Kutuları yığında ayrılır ve atık olarak toplanmış, çok fazla kutulama ve kutudan çıkarma nesneler olduğundan, yığın, atık toplayıcı ve nihai olarak uygulamanın performansı üzerinde olumsuz bir etkiye sahip olabilir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-114">Because boxes are objects that are allocated on the heap and are garbage-collected, too much boxing and unboxing can have a negative impact on the heap, the garbage collector, and ultimately the performance of the application.</span></span>  <span data-ttu-id="8df5a-115">Buna karşılık, başvuru türleri dönüştürme gibi böyle bir kutulama gerçekleşir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-115">In contrast, no such boxing occurs as reference types are cast.</span></span>  
  
 <span data-ttu-id="8df5a-116">Ardından, değer türü atamaları tüm değeri kopyalayın ise başvuru türü atamaları başvuru kopyalayın.</span><span class="sxs-lookup"><span data-stu-id="8df5a-116">Next, reference type assignments copy the reference, whereas value type assignments copy the entire value.</span></span> <span data-ttu-id="8df5a-117">Bu nedenle, büyük referans tür atamaları büyük değer türlerinin atamaları ucuz.</span><span class="sxs-lookup"><span data-stu-id="8df5a-117">Therefore, assignments of large reference types are cheaper than assignments of large value types.</span></span>  
  
 <span data-ttu-id="8df5a-118">Son olarak, değer türleri değere göre geçirilir ancak başvuru türleri başvuruya göre iletilir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-118">Finally, reference types are passed by reference, whereas value types are passed by value.</span></span> <span data-ttu-id="8df5a-119">Bir başvuru türünün örneğini yapılan değişiklikler tüm başvuruları örneğine işaret etkiler.</span><span class="sxs-lookup"><span data-stu-id="8df5a-119">Changes to an instance of a reference type affect all references pointing to the instance.</span></span> <span data-ttu-id="8df5a-120">Değer türü örnekleri değere göre geçirildiğinde kopyalanır.</span><span class="sxs-lookup"><span data-stu-id="8df5a-120">Value type instances are copied when they are passed by value.</span></span> <span data-ttu-id="8df5a-121">Bir değer türü örneği değiştirildiğinde, Elbette herhangi birini kendi kopya etkilemez.</span><span class="sxs-lookup"><span data-stu-id="8df5a-121">When an instance of a value type is changed, it of course does not affect any of its copies.</span></span> <span data-ttu-id="8df5a-122">Kopya açıkça kullanıcı tarafından oluşturulmaz, ancak bağımsız değişkenler geçirilir veya dönüş değerleri döndürülür, örtük olarak oluşturulur, değiştirilebilir bir değer türleri birçok kullanıcılar için kafa karıştırıcı olabilir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-122">Because the copies are not created explicitly by the user but are implicitly created when arguments are passed or return values are returned, value types that can be changed can be confusing to many users.</span></span> <span data-ttu-id="8df5a-123">Bu nedenle, değer türleri sabit olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="8df5a-123">Therefore, value types should be immutable.</span></span>  
  
 <span data-ttu-id="8df5a-124">Bir kural karşısında, bir Framework türlerinin çoğu sınıflar olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="8df5a-124">As a rule of thumb, the majority of types in a framework should be classes.</span></span> <span data-ttu-id="8df5a-125">Ancak, bir değer türü özelliklerini yapıları kullanmak daha uygun hale bazı durumlar vardır.</span><span class="sxs-lookup"><span data-stu-id="8df5a-125">There are, however, some situations in which the characteristics of a value type make it more appropriate to use structs.</span></span>  
  
 <span data-ttu-id="8df5a-126">**✓ CONSIDER** türünün örnekleri küçük ve yaygın olarak kısa süreli veya diğer nesneleri genellikle katıştırılmış yapı yerine bir sınıf tanımlama.</span><span class="sxs-lookup"><span data-stu-id="8df5a-126">**✓ CONSIDER** defining a struct instead of a class if instances of the type are small and commonly short-lived or are commonly embedded in other objects.</span></span>  
  
 <span data-ttu-id="8df5a-127">**X AVOID** türü tüm aşağıdaki özelliklere sahip olmadığı sürece yapı tanımlama:</span><span class="sxs-lookup"><span data-stu-id="8df5a-127">**X AVOID** defining a struct unless the type has all of the following characteristics:</span></span>  
  
-   <span data-ttu-id="8df5a-128">Mantıksal olarak, ilkel türler için benzer tek bir değeri temsil eder (`int`, `double`vb..).</span><span class="sxs-lookup"><span data-stu-id="8df5a-128">It logically represents a single value, similar to primitive types (`int`, `double`, etc.).</span></span>  
  
-   <span data-ttu-id="8df5a-129">Bir örnek boyutu altında 16 bayt var.</span><span class="sxs-lookup"><span data-stu-id="8df5a-129">It has an instance size under 16 bytes.</span></span>  
  
-   <span data-ttu-id="8df5a-130">Sabittir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-130">It is immutable.</span></span>  
  
-   <span data-ttu-id="8df5a-131">Bu sık Kutulu gerekmez.</span><span class="sxs-lookup"><span data-stu-id="8df5a-131">It will not have to be boxed frequently.</span></span>  
  
 <span data-ttu-id="8df5a-132">Diğer tüm durumlarda, türlerinizi sınıfları olarak tanımlamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="8df5a-132">In all other cases, you should define your types as classes.</span></span>
