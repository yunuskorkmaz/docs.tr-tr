# <a name="property-design"></a><span data-ttu-id="4935d-102">Özellik tasarımı</span><span class="sxs-lookup"><span data-stu-id="4935d-102">Property Design</span></span>
<span data-ttu-id="4935d-103">Özellikleri teknik yöntemlere benzer olsa da, bunlar kendi kullanım senaryoları açısından oldukça farklıdır.</span><span class="sxs-lookup"><span data-stu-id="4935d-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="4935d-104">Bunlar, akıllı alanları olarak başlatmasında gösterilmelidir.</span><span class="sxs-lookup"><span data-stu-id="4935d-104">They should be seen as smart fields.</span></span> <span data-ttu-id="4935d-105">Alanları arama söz dizimi ve yöntemleri esnekliğini sahiptirler.</span><span class="sxs-lookup"><span data-stu-id="4935d-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>  
  
 <span data-ttu-id="4935d-106">**✓ DO** çağıran özelliğin değerini değiştirmek açmaması gereken, yalnızca get özellikleri oluşturun.</span><span class="sxs-lookup"><span data-stu-id="4935d-106">**✓ DO** create get-only properties if the caller should not be able to change the value of the property.</span></span>  
  
 <span data-ttu-id="4935d-107">Türü olmadığını aklınızda bulundurun özelliği kesilebilir başvuru türü, özellik değeri salt alma özelliği olsa bile değiştirilebilir.</span><span class="sxs-lookup"><span data-stu-id="4935d-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>  
  
 <span data-ttu-id="4935d-108">**X DO NOT** alıcı daha geniş erişilebilirlik sahip ayarlayıcı yalnızca küme özellikleri veya özellikleri sağlar.</span><span class="sxs-lookup"><span data-stu-id="4935d-108">**X DO NOT** provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>  
  
 <span data-ttu-id="4935d-109">Örneğin, özellikler, genel bir Ayarlayıcısı ve korumalı bir alıcı ile kullanmayın.</span><span class="sxs-lookup"><span data-stu-id="4935d-109">For example, do not use properties with a public setter and a protected getter.</span></span>  
  
 <span data-ttu-id="4935d-110">Özellik Get yordamı sağlanan, bir yöntem işlevselliği bunun yerine uygulayın.</span><span class="sxs-lookup"><span data-stu-id="4935d-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="4935d-111">Yöntem adı ile başlamayı düşünün `Set` ne özelliği adlı ile izleyin.</span><span class="sxs-lookup"><span data-stu-id="4935d-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="4935d-112">Örneğin, <xref:System.AppDomain> adlı bir yöntem olan `SetCachePath` adlı yalnızca kümesi özelliği yerine `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="4935d-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>  
  
 <span data-ttu-id="4935d-113">**✓ DO** tüm özellikleri, varsayılan bir güvenlik açığı veya çok verimsiz kodu yol açmamasını sağlamak için duyarlı varsayılan değerler sağlayın.</span><span class="sxs-lookup"><span data-stu-id="4935d-113">**✓ DO** provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>  
  
 <span data-ttu-id="4935d-114">**✓ DO** bu nesne bir geçici geçersiz durumda sonuçları olsa bile herhangi bir sırada ayarlanacak özellikler izin verir.</span><span class="sxs-lookup"><span data-stu-id="4935d-114">**✓ DO** allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>  
  
 <span data-ttu-id="4935d-115">Aynı nesne üzerinde diğer özelliklerin değerlerine burada bazı özellik değerlerini geçersiz olabilir bir noktaya birbiriyle ilişkili iki veya daha fazla özelliklerini yaygındır.</span><span class="sxs-lookup"><span data-stu-id="4935d-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="4935d-116">Birbiriyle ilişkili özellikleri gerçekten birlikte nesne tarafından kullanılan kadar bu gibi durumlarda, geçersiz durumdan kaynaklanan özel durumları ertelenebiliyorsa.</span><span class="sxs-lookup"><span data-stu-id="4935d-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>  
  
 <span data-ttu-id="4935d-117">**✓ DO** özellik set yordamı bir özel durum oluşturursa önceki değeri korumak.</span><span class="sxs-lookup"><span data-stu-id="4935d-117">**✓ DO** preserve the previous value if a property setter throws an exception.</span></span>  
  
 <span data-ttu-id="4935d-118">**X AVOID** özellik alıcıları özel durumları atma.</span><span class="sxs-lookup"><span data-stu-id="4935d-118">**X AVOID** throwing exceptions from property getters.</span></span>  
  
 <span data-ttu-id="4935d-119">Özellik alıcılar basit işlemler olmalıdır ve tüm önkoşullara sahip olmamalıdır.</span><span class="sxs-lookup"><span data-stu-id="4935d-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="4935d-120">Bir alıcı bir özel durum oluşturabilecek, bir yöntem için büyük olasılıkla tasarlanması gerekir.</span><span class="sxs-lookup"><span data-stu-id="4935d-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="4935d-121">Bu kural, dizin oluşturucular, bağımsız değişkenleri doğrulamak sonucunda özel durumlar nerede bekliyoruz uygulanmaz dikkat edin.</span><span class="sxs-lookup"><span data-stu-id="4935d-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>  
  
### <a name="indexed-property-design"></a><span data-ttu-id="4935d-122">Dizinlenmiş özellik tasarımı</span><span class="sxs-lookup"><span data-stu-id="4935d-122">Indexed Property Design</span></span>  
 <span data-ttu-id="4935d-123">Dizini oluşturulmuş özelliğe parametrelerine sahip olabilir ve dizi dizini oluşturma için benzer bir özel sözdizimi ile çağrılabilen özel bir özelliktir.</span><span class="sxs-lookup"><span data-stu-id="4935d-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>  
  
 <span data-ttu-id="4935d-124">Dizinli Özellikler, dizin oluşturucular yaygın olarak adlandırılır.</span><span class="sxs-lookup"><span data-stu-id="4935d-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="4935d-125">Dizin oluşturucular mantıksal bir koleksiyondaki öğelerin erişim sağlayan API kullanılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="4935d-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="4935d-126">Örneğin, bir dizi karakter ve dizin oluşturucusunda bir dize olan <xref:System.String?displayProperty=nameWithType> kendi karakterlerin erişmek için eklendi.</span><span class="sxs-lookup"><span data-stu-id="4935d-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>  
  
 <span data-ttu-id="4935d-127">**✓ CONSIDER** iç dizisinde depolanan verilere erişim sağlamak için dizin oluşturucular kullanma.</span><span class="sxs-lookup"><span data-stu-id="4935d-127">**✓ CONSIDER** using indexers to provide access to data stored in an internal array.</span></span>  
  
 <span data-ttu-id="4935d-128">**✓ CONSIDER** öğe koleksiyonunu temsil eden türlerinde dizin oluşturucular sağlama.</span><span class="sxs-lookup"><span data-stu-id="4935d-128">**✓ CONSIDER** providing indexers on types representing collections of items.</span></span>  
  
 <span data-ttu-id="4935d-129">**X AVOID** dizin oluşturulmuş özellikleri birden fazla parametresiyle kullanarak.</span><span class="sxs-lookup"><span data-stu-id="4935d-129">**X AVOID** using indexed properties with more than one parameter.</span></span>  
  
 <span data-ttu-id="4935d-130">Tasarım birden çok parametre gerektiriyorsa, özelliğin gerçekten mantıksal bir koleksiyon için bir erişimci temsil edip etmediğini yeniden belirleyin.</span><span class="sxs-lookup"><span data-stu-id="4935d-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="4935d-131">Kullanmıyorsa, bunun yerine yöntemlerini kullanın.</span><span class="sxs-lookup"><span data-stu-id="4935d-131">If it does not, use methods instead.</span></span> <span data-ttu-id="4935d-132">Yöntem adı ile başlamayı düşünün `Get` veya `Set`.</span><span class="sxs-lookup"><span data-stu-id="4935d-132">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="4935d-133">**X AVOID** dışında parametre türleri oluşturucularla <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, veya enum.</span><span class="sxs-lookup"><span data-stu-id="4935d-133">**X AVOID** indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>  
  
 <span data-ttu-id="4935d-134">Diğer parametre türleri tasarımı gerektiriyorsa, API, gerçekten erişimci mantıksal bir koleksiyonu temsil edip etmediğini kesinlikle yeniden değerlendir.</span><span class="sxs-lookup"><span data-stu-id="4935d-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="4935d-135">Kullanmıyorsa, bir yöntem kullanın.</span><span class="sxs-lookup"><span data-stu-id="4935d-135">If it does not, use a method.</span></span> <span data-ttu-id="4935d-136">Yöntem adı ile başlamayı düşünün `Get` veya `Set`.</span><span class="sxs-lookup"><span data-stu-id="4935d-136">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="4935d-137">**✓ DO** adını kullanın `Item` için tabii daha iyi bir adı olmadıkça özellikleri dizine (örn., bkz: <xref:System.String.Chars%2A> özellikte `System.String`).</span><span class="sxs-lookup"><span data-stu-id="4935d-137">**✓ DO** use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>  
  
 <span data-ttu-id="4935d-138">C# ' ta adlı öğe varsayılan dizin oluşturucular özelliklere.</span><span class="sxs-lookup"><span data-stu-id="4935d-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="4935d-139"><xref:System.Runtime.CompilerServices.IndexerNameAttribute> Bu ad özelleştirmek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="4935d-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>  
  
 <span data-ttu-id="4935d-140">**X DO NOT** hem bir dizin oluşturucu hem de anlam olarak eşdeğerdir yöntemleri sağlar.</span><span class="sxs-lookup"><span data-stu-id="4935d-140">**X DO NOT** provide both an indexer and methods that are semantically equivalent.</span></span>  
  
 <span data-ttu-id="4935d-141">**X DO NOT** aşırı yüklenmiş dizin oluşturucular bir türde birden fazla ailesi sağlayın.</span><span class="sxs-lookup"><span data-stu-id="4935d-141">**X DO NOT** provide more than one family of overloaded indexers in one type.</span></span>  
  
 <span data-ttu-id="4935d-142">Bu, C# Derleyici tarafından zorlanır.</span><span class="sxs-lookup"><span data-stu-id="4935d-142">This is enforced by the C# compiler.</span></span>  
  
 <span data-ttu-id="4935d-143">**X DO NOT** kullanım varsayılan olmayan dizin oluşturulmuş özellikleri.</span><span class="sxs-lookup"><span data-stu-id="4935d-143">**X DO NOT** use nondefault indexed properties.</span></span>  
  
 <span data-ttu-id="4935d-144">Bu, C# Derleyici tarafından zorlanır.</span><span class="sxs-lookup"><span data-stu-id="4935d-144">This is enforced by the C# compiler.</span></span>  
  
### <a name="property-change-notification-events"></a><span data-ttu-id="4935d-145">Özellik değişikliği bildirimi olayları</span><span class="sxs-lookup"><span data-stu-id="4935d-145">Property Change Notification Events</span></span>  
 <span data-ttu-id="4935d-146">Bazı durumlarda kullanıcı bir özellik değeri değişiklikleri bildiren bir olay sağlamak kullanışlıdır.</span><span class="sxs-lookup"><span data-stu-id="4935d-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="4935d-147">Örneğin, `System.Windows.Forms.Control` başlatır bir `TextChanged` olayından sonra değerini kendi `Text` özelliği değişti.</span><span class="sxs-lookup"><span data-stu-id="4935d-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>  
  
 <span data-ttu-id="4935d-148">**✓ CONSIDER** değişiklik üst düzey API'leri (genellikle Tasarımcı bileşenleri) özellik değerlerinde değişiklik yapıldığında bildirim olayları oluşturma.</span><span class="sxs-lookup"><span data-stu-id="4935d-148">**✓ CONSIDER** raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>  
  
 <span data-ttu-id="4935d-149">Bir nesnenin bir özelliğini ne zaman değişiyor bilmek, bir kullanıcı için iyi bir senaryo ise, nesne özelliği için bir değişikliği bildirim olayı harekete geçirmelidir.</span><span class="sxs-lookup"><span data-stu-id="4935d-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>  
  
 <span data-ttu-id="4935d-150">Ancak, bu tür taban türler veya koleksiyonları gibi alt düzey API'ler için olay yükü değer olması beklenmez.</span><span class="sxs-lookup"><span data-stu-id="4935d-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="4935d-151">Örneğin, <xref:System.Collections.Generic.List%601> yeni bir öğe listesine eklendiğinde bu tür olayların oluşturmaz ve `Count` özellik değişiklikleri.</span><span class="sxs-lookup"><span data-stu-id="4935d-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>  
  
 <span data-ttu-id="4935d-152">**✓ CONSIDER** değişiklik bir özelliğin değerini dış zorlar değiştiğinde bildirim olayları oluşturma.</span><span class="sxs-lookup"><span data-stu-id="4935d-152">**✓ CONSIDER** raising change notification events when the value of a property changes via external forces.</span></span>  
  
 <span data-ttu-id="4935d-153">Bazı dış zorla (bir yolla nesne üzerinde yöntemleri çağırarak dışında) aracılığıyla bir özellik değeri değişirse yükseltmek olayları göstermek için geliştirici değer değişiyor ve değişti.</span><span class="sxs-lookup"><span data-stu-id="4935d-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="4935d-154">İyi bir örnektir `Text` metin kutusu denetimi özelliği.</span><span class="sxs-lookup"><span data-stu-id="4935d-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="4935d-155">Kullanıcının yazdığı metni ne zaman bir `TextBox`, özellik değeri otomatik olarak değiştirir.</span><span class="sxs-lookup"><span data-stu-id="4935d-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
