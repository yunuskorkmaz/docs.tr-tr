<span data-ttu-id="89cc0-108">Şekil 4-20 yalnızca bir tek parçalı bir istemci uygulamasından mikro hizmetler kullanan basit bir yaklaşım gösterilmektedir.</span><span class="sxs-lookup"><span data-stu-id="89cc0-108">Figure 4-20 shows the simpler approach of just consuming microservices from a monolithic client application.</span></span> <span data-ttu-id="89cc0-109">Elbette, HTML ve JavaScript üretme aralığındaki bir ASP.NET MVC hizmeti olabilir.</span><span class="sxs-lookup"><span data-stu-id="89cc0-109">Of course, you could have an ASP.NET MVC service in between producing the HTML and JavaScript.</span></span> <span data-ttu-id="89cc0-110">Şekilde, tek bir (parçalı) istemci kullanıcı Arabirimi mantığı ve verileri ve kullanıcı Arabirimi şekli (HTML ve JavaScript) değil, yalnızca odak mikro hizmetler kullanan olduğunu vurgular bir basitleştirme ' dir.</span><span class="sxs-lookup"><span data-stu-id="89cc0-110">The figure is a simplification that highlights that you have a single (monolithic) client UI consuming the microservices, which just focus on logic and data and not on the UI shape (HTML and JavaScript).</span></span>

<span data-ttu-id="89cc0-116">İstemci uygulama başlangıç zamanında her istemci kullanıcı Arabirimi bileşenleri (örneğin, TypeScript sınıflar) kendini Viewmodel'lar belirli bir senaryo için sağlayabilen bir altyapı mikro ile kaydeder.</span><span class="sxs-lookup"><span data-stu-id="89cc0-116">At client application start-up time, each of the client UI components (TypeScript classes, for example) registers itself with an infrastructure microservice capable of providing ViewModels for a given scenario.</span></span> <span data-ttu-id="89cc0-117">Mikro hizmet şekli değişirse, kullanıcı arabirimini de değişir.</span><span class="sxs-lookup"><span data-stu-id="89cc0-117">If the microservice changes the shape, the UI changes also.</span></span>

<span data-ttu-id="89cc0-118">Şekil 4-21, bu bileşik kullanıcı Arabirimi yaklaşım bir sürümünü gösterir.</span><span class="sxs-lookup"><span data-stu-id="89cc0-118">Figure 4-21 shows a version of this composite UI approach.</span></span> <span data-ttu-id="89cc0-119">Farklı teknikleri üzerinde göre ayrıntılı bölümleri toplayarak diğer mikro hizmetler olabileceği için bu basitleştirilmiştir —, geleneksel web yaklaşım (ASP.NET MVC) veya bir SPA (tek sayfalı uygulama) oluşturma üzerine bağlıdır.</span><span class="sxs-lookup"><span data-stu-id="89cc0-119">This is simplified, because you might have other microservices that are aggregating granular parts based on different techniques—it depends on whether you are building a traditional web approach (ASP.NET MVC) or an SPA (Single Page Application).</span></span>

<span data-ttu-id="89cc0-124">Mikro hizmet tarafından yönetilen bir bileşik kullanıcı Arabirimi yaklaşım daha zor olabilir ya da daha az bir şekilde bağlı olarak hangi UI teknolojileri kullanıyorsanız.</span><span class="sxs-lookup"><span data-stu-id="89cc0-124">A composite UI approach that is driven by microservices can be more challenging or less so, depending on what UI technologies you are using.</span></span> <span data-ttu-id="89cc0-125">Örneğin, aynı teknikleri, yerel mobil uygulama veya bir SPA oluşturmak için kullandığınız bir geleneksel web uygulaması oluşturmak için kullanacağınız değil (olarak için bu yaklaşım daha zor olabilecek Xamarin uygulamaları geliştirirken).</span><span class="sxs-lookup"><span data-stu-id="89cc0-125">For instance, you will not use the same techniques for building a traditional web application that you use for building an SPA or for native mobile app (as when developing Xamarin apps, which can be more challenging for this approach).</span></span>

<span data-ttu-id="89cc0-126">[Hizmetine](https://aka.ms/MicroservicesArchitecture) örnek uygulama, birden çok nedenlerle tek parça UI yaklaşımı kullanır.</span><span class="sxs-lookup"><span data-stu-id="89cc0-126">The [eShopOnContainers](https://aka.ms/MicroservicesArchitecture) sample application uses the monolithic UI approach for multiple reasons.</span></span> <span data-ttu-id="89cc0-127">İlk olarak, mikro hizmetler ve kapsayıcılar giriş niteliğindedir.</span><span class="sxs-lookup"><span data-stu-id="89cc0-127">First, it is an introduction to microservices and containers.</span></span> <span data-ttu-id="89cc0-128">Bir bileşik kullanıcı Arabirimi daha gelişmiş ancak aynı zamanda daha fazla tasarlama ve kullanıcı arabirimini geliştirirken karmaşıklığı gerektirir.</span><span class="sxs-lookup"><span data-stu-id="89cc0-128">A composite UI is more advanced but also requires further complexity when designing and developing the UI.</span></span> <span data-ttu-id="89cc0-129">Ayrıca, daha karmaşık C istemcide yapacağınız Xamarin, temel yerel bir mobil uygulama ikinci hizmetine sağlar\# yan.</span><span class="sxs-lookup"><span data-stu-id="89cc0-129">Second, eShopOnContainers also provides a native mobile app based on Xamarin, which would make it more complex on the client C\# side.</span></span>
