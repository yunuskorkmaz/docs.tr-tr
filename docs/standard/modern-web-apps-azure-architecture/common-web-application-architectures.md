## <a name="what-are-layers"></a><span data-ttu-id="cddfe-132">Katmanları nelerdir?</span><span class="sxs-lookup"><span data-stu-id="cddfe-132">What are layers?</span></span>

<span data-ttu-id="cddfe-133">Uygulamaların karmaşıklığı arttıkça, bu karmaşıklık yönetmek için bir uygulamanın kendi sorumluluklarını veya endişeniz varsa göre bölüneceği yoludur.</span><span class="sxs-lookup"><span data-stu-id="cddfe-133">As applications grow in complexity, one way to manage that complexity is to break up the application according to its responsibilities or concerns.</span></span> <span data-ttu-id="cddfe-134">Bu sorunları asıl ayrımı izler ve böylece geliştiriciler, burada bazı işlevlere uygulanan kolayca bulabilirsiniz düzenlenmiş büyüyen bir kod temeli korunmasına yardımcı olabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="cddfe-134">This follows the separation of concerns principle, and can help keep a growing codebase organized so that developers can easily find where certain functionality is implemented.</span></span> <span data-ttu-id="cddfe-135">Katmanlı bir mimari, çok sayıda avantaj yalnızca kod kuruluş dışına yine de sunar.</span><span class="sxs-lookup"><span data-stu-id="cddfe-135">Layered architecture offers a number of advantages beyond just code organization, though.</span></span>

<span data-ttu-id="cddfe-136">Kod katmanlara düzenleyerek, düşük düzey ortak işlevselliği uygulama genelinde yeniden kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="cddfe-136">By organizing code into layers, common low-level functionality can be reused throughout the application.</span></span> <span data-ttu-id="cddfe-137">Bu yeniden yazılması gerekiyor daha az kod anlamına geldiğinden ve aşağıdaki tek bir uygulama üzerinde standart hale getirmek uygulama izin yararlıdır [kendiniz (KURU) yineleme](https://en.wikipedia.org/wiki/Don%27t_repeat_yourself) ilkesi.</span><span class="sxs-lookup"><span data-stu-id="cddfe-137">This reuse is beneficial because it means less code needs to be written and because it can allow the application to standardize on a single implementation, following the [don't repeat yourself (DRY)](https://en.wikipedia.org/wiki/Don%27t_repeat_yourself) principle.</span></span>
