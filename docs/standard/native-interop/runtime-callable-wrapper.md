# <a name="runtime-callable-wrapper"></a><span data-ttu-id="d96cc-103">Çalışma Zamanı Aranabilir Sarmalayıcısı</span><span class="sxs-lookup"><span data-stu-id="d96cc-103">Runtime Callable Wrapper</span></span>
<span data-ttu-id="d96cc-104">Ortak dil çalışma zamanı, COM nesnelerini, çalışma zamanı çağrılabilir sarmalayıcı (RCW) adlı bir ara sunucu üzerinden kullanıma sunar.</span><span class="sxs-lookup"><span data-stu-id="d96cc-104">The common language runtime exposes COM objects through a proxy called the runtime callable wrapper (RCW).</span></span> <span data-ttu-id="d96cc-105">RCW, .NET istemcilerine sıradan bir nesne gibi görünse de, birincil işlevi bir .NET istemcisiyle COM nesnesi arasındaki çağrıları sıralayamaz.</span><span class="sxs-lookup"><span data-stu-id="d96cc-105">Although the RCW appears to be an ordinary object to .NET clients, its primary function is to marshal calls between a .NET client and a COM object.</span></span>  
  
 <span data-ttu-id="d96cc-106">Çalışma zamanı, bu nesnede bulunan başvuruların sayısından bağımsız olarak her bir COM nesnesi için tam olarak bir RCW oluşturur.</span><span class="sxs-lookup"><span data-stu-id="d96cc-106">The runtime creates exactly one RCW for each COM object, regardless of the number of references that exist on that object.</span></span> <span data-ttu-id="d96cc-107">Çalışma zamanı her bir nesne için işlem başına tek bir RCW tutar.</span><span class="sxs-lookup"><span data-stu-id="d96cc-107">The runtime maintains a single RCW per process for each object.</span></span>  <span data-ttu-id="d96cc-108">Bir uygulama etki alanında veya grupta bir RCW oluşturup daha sonra başka bir uygulama etki alanına veya gruba bir başvuru geçirirseniz, ilk nesneye yönelik bir ara sunucu kullanılacaktır.</span><span class="sxs-lookup"><span data-stu-id="d96cc-108">If you create an RCW in one application domain or apartment, and then pass a reference to another application domain or apartment, a proxy to the first object will be used.</span></span>  <span data-ttu-id="d96cc-109">Aşağıdaki çizimde gösterildiği gibi, herhangi bir sayıda yönetilen istemci, INEW ve INewer arabirimlerini sunan COM nesnelerine bir başvuru tutabilir.</span><span class="sxs-lookup"><span data-stu-id="d96cc-109">As the following illustration shows, any number of managed clients can hold a reference to the COM objects that expose INew and INewer interfaces.</span></span>  

<span data-ttu-id="d96cc-110">Aşağıdaki görüntüde, çalışma zamanında çağrılabilir sarmalayıcı aracılığıyla COM nesnelerine erişme işlemi gösterilmektedir:</span><span class="sxs-lookup"><span data-stu-id="d96cc-110">The following image shows the process for accessing COM objects through the runtime callable wrapper:</span></span>

 <span data-ttu-id="d96cc-112">Bir tür kitaplığından türetilmiş meta verileri kullanarak, çalışma zamanı hem çağrılan COM nesnesini hem de bu nesne için bir sarmalayıcı oluşturur.</span><span class="sxs-lookup"><span data-stu-id="d96cc-112">Using metadata derived from a type library, the runtime creates both the COM object being called and a wrapper for that object.</span></span> <span data-ttu-id="d96cc-113">Her RCW, nesne sarmaladığı COM nesnesinde bir arabirim işaretçileri önbelleği tutar ve RCW artık gerekmiyorsa COM nesnesinde başvurusunu yayınlar.</span><span class="sxs-lookup"><span data-stu-id="d96cc-113">Each RCW maintains a cache of interface pointers on the COM object it wraps and releases its reference on the COM object when the RCW is no longer needed.</span></span> <span data-ttu-id="d96cc-114">Çalışma zamanı RCW üzerinde çöp toplama işlemi gerçekleştirir.</span><span class="sxs-lookup"><span data-stu-id="d96cc-114">The runtime performs garbage collection on the RCW.</span></span>  
  
 <span data-ttu-id="d96cc-115">Diğer etkinliklerin yanı sıra, RCW, Sarmalanan nesne adına yönetilen ve yönetilmeyen kod arasında veri sıraladığında.</span><span class="sxs-lookup"><span data-stu-id="d96cc-115">Among other activities, the RCW marshals data between managed and unmanaged code, on behalf of the wrapped object.</span></span> <span data-ttu-id="d96cc-116">Özellikle, RCW, istemci ve sunucu aralarında geçen verilerin farklı temsillerine sahip olduğunda, yöntem bağımsız değişkenleri ve yöntem dönüş değerleri için sıralama sağlar.</span><span class="sxs-lookup"><span data-stu-id="d96cc-116">Specifically, the RCW provides marshaling for method arguments and method return values whenever the client and server have different representations of the data passed between them.</span></span>  
  
 <span data-ttu-id="d96cc-117">Standart sarmalayıcı, yerleşik sıralama kurallarını uygular.</span><span class="sxs-lookup"><span data-stu-id="d96cc-117">The standard wrapper enforces built-in marshaling rules.</span></span> <span data-ttu-id="d96cc-118">Örneğin, bir .NET istemcisi bir dize türünü yönetilmeyen bir nesneye bir bağımsız değişkenin parçası olarak geçirdiğinde, sarmalayıcı dizeyi bir BSTR türüne dönüştürür.</span><span class="sxs-lookup"><span data-stu-id="d96cc-118">For example, when a .NET client passes a String type as part of an argument to an unmanaged object, the wrapper converts the string to a BSTR type.</span></span> <span data-ttu-id="d96cc-119">COM nesnesi, yönetilen çağıranına bir BSTR döndürmelidir, çağıran bir dize alır.</span><span class="sxs-lookup"><span data-stu-id="d96cc-119">Should the COM object return a BSTR to its managed caller, the caller receives a String.</span></span> <span data-ttu-id="d96cc-120">Hem istemci hem de sunucu onlara tanıdık gelen verileri gönderir ve alır.</span><span class="sxs-lookup"><span data-stu-id="d96cc-120">Both the client and the server send and receive data that is familiar to them.</span></span> <span data-ttu-id="d96cc-121">Diğer türler dönüştürme gerektirmez.</span><span class="sxs-lookup"><span data-stu-id="d96cc-121">Other types require no conversion.</span></span> <span data-ttu-id="d96cc-122">Örneğin, standart bir sarmalayıcı, türü dönüştürmeden yönetilen ve yönetilmeyen kod arasında her zaman 4 baytlık bir tamsayı geçilecektir.</span><span class="sxs-lookup"><span data-stu-id="d96cc-122">For instance, a standard wrapper will always pass a 4-byte integer between managed and unmanaged code without converting the type.</span></span>
